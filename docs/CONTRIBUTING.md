# Contributing to Swift Utilities Contract

Thank you for your interest in contributing! Here are some guidelines to help you get started.

## Development Environment
1. Clone the repository
2. Copoy `.env.example` to `.env` and fill in necessary values
3. Install dependencies: `npm install`
4. Run tests to ensure everything is working: `npm test`

## Project Structure
- `contracts/`: Smart contracts (Solidity)
- `scripts/`: Deployment and operation scripts
- `test/`: Hardhat tests

## Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes (`npm test`).
4. Format your code using the project's style.

## Coding Standards
- Use solidity 0.8.x
- Follow standard smart contract security practices
- Use Natspec comments for contract documentation

## Issues
- Use the issue tracker to report bugs or suggest features.
- Please be descriptive and provide reproduction steps for bugs.
